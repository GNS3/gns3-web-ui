## 升级策略

### 方案一：直接升级

```bash
npm install @angular/core@19 @angular/material@19 --legacy-peer-deps
```

**优点**: 快速
**缺点**: 可能遇到运行时错误

### 方案二：逐个升级

1. 先升级 Angular 核心包
2. 逐个升级第三方库
3. 运行测试确认

**优点**: 更容易定位问题
**缺点**: 时间较长

### 方案三：使用 ng update

```bash
npx @angular/cli@19 update @angular/core@19 @angular/material@19
```

**优点**: 自动迁移
**缺点**: 可能不完整

---

## 建议

1. **先备份**: 创建新分支进行升级
2. **使用 --legacy-peer-deps**: 避免 peer dependency 冲突
3. **测试**: 升级后运行完整测试
4. **手动修复**: 准备好手动修改不兼容的库
