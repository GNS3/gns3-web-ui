# Angular 17 → 18 升级分析

## 详细变更分析

### 1. TypeScript 版本要求

**当前**: TypeScript 5.4.x
**需要**: TypeScript 5.5+

**修改文件**:
- `package.json` - 更新 typescript 版本

```json
// 修改前
"typescript": "~5.4.0"

// 修改后
"typescript": "~5.5.0"
```

### 4. Forms 使用情况（无需修改）

**67 个文件** 使用 FormControl/FormGroup：
- Angular 18 对 Forms 类型有改进，但向后兼容
- 现有代码无需修改

---

### 5. NgModules（无需修改）

**5 个 NgModule**：
- `src/app/app.module.ts`
- `src/app/app-routing.module.ts`
- `src/app/cartography/cartography.module.ts`
- `src/app/testing/app-testing/app-testing.module.ts`

NgModule 在 Angular 18 中仍然完全支持，无需修改。

---

### 6. Zone.js

**当前**: `zone.js": "^0.11.5"`

Angular 18 仍然支持 Zone.js，无需修改。

---

### 7. 主题系统

**文件**: `src/styles/theme-v2.scss`

Angular 18 对 Material 主题 API 有以下变化：
- 继续使用 `define-palette` 和 `define-theme`
- 可能需要更新到新版本

## 预期风险

### 低风险
- TypeScript 小版本升级
- Angular 核心包增量更新

### 中风险
- 第三方库兼容性（ngx-markdown, ng2-file-upload 等）
- 可能需要 `--legacy-peer-deps`

### 无风险
- Forms 类型（向后兼容）
- NgModule（完全支持）
- Zone.js（默认使用）

---

## 升级命令

```bash
# 1. 更新 package.json 中的版本

# 2. 安装新版本
npm install --legacy-peer-deps

# 3. 构建测试
npm run build

# 4. 运行测试
npm test
```

---

## 总结

Angular 17 → 18 是一个相对平滑的升级：

1. **主要变更**: TypeScript 5.5+ 要求
2. **无需大规模重构**: Forms 和 NgModule 仍然兼容
3. **可选迁移**: 新的控制流语法（@if/@for）可以使用旧语法
4. **风险评估**: 低风险升级
