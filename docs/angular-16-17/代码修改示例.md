# 代码修改示例

## 概述

本文档提供 Angular 16 到 Angular 17 升级过程中需要修改的代码示例。

---

## 1. TypeScript 版本更新

### package.json

```json
// 修改前
"typescript": "~4.9.5"

// 修改后
"typescript": "~5.0.0"
```

---

## 2. Forms 类型修复

### 2.1 使用 UntypedFormControl

```typescript
// 修改前 (Angular 16)
import { FormControl, FormGroup } from '@angular/forms';

export class ExampleComponent {
  name = new FormControl('');
  age = new FormControl(0);
  form = new FormGroup({
    name: new FormControl(''),
    age: new FormControl(0)
  });
}
```

```typescript
// 修改后 (Angular 17)
import { UntypedFormControl, UntypedFormGroup } from '@angular/forms';

export class ExampleComponent {
  name = new UntypedFormControl('');
  age = new UntypedFormControl(0);
  form = new UntypedFormGroup({
    name: new UntypedFormControl(''),
    age: new UntypedFormControl(0)
  });
}
```

### 2.2 使用泛型类型

```typescript
// 修改后 (Angular 17) - 使用泛型
import { FormControl, FormGroup } from '@angular/forms';

export class ExampleComponent {
  name = new FormControl<string>('');
  age = new FormControl<number>(0);
  form = new FormGroup<{ name: FormControl<string>; age: FormControl<number> }>({
    name: new FormControl<string>(''),
    age: new FormControl<number>(0)
  });
}
```

### 2.3 在 HTML 模板中

```html
<!-- 修改前 -->
<input [formControl]="name">

<!-- 修改后 -->
<input [formControl]="name">
<!-- 或使用新的 @if 语法 -->
@if (name.value) {
  <input [formControl]="name">
}
```

## 3. 新的控制流语法（可选）

### 3.1 @if 替换 *ngIf

```html
<!-- 修改前 -->
<div *ngIf="isVisible">内容</div>

<!-- 修改后（推荐） -->
@if (isVisible) {
  <div>内容</div>
}
```

### 3.2 @for 替换 *ngFor

```html
<!-- 修改前 -->
<div *ngFor="let item of items">{{ item.name }}</div>

<!-- 修改后（推荐） -->
@for (item of items; track item.id) {
  <div>{{ item.name }}</div>
}
```

### 3.3 @switch 替换 ngSwitch

```html
<!-- 修改前 -->
<div [ngSwitch]="type">
  <div *ngSwitchCase="'a'">Type A</div>
  <div *ngSwitchCase="'b'">Type B</div>
  <div *ngSwitchDefault>Default</div>
</div>

<!-- 修改后（推荐） -->
@switch (type) {
  @case ('a') { <div>Type A</div> }
  @case ('b') { <div>Type B</div> }
  @default { <div>Default</div> }
}
```

## 4. inject() 函数（推荐）

### 4.1 构造函数注入

```typescript
// 修改前
constructor(private service: MyService) {}

// 修改后（推荐）
private service = inject(MyService);
```

### 4.2 完整示例

```typescript
// 修改前
@Component({...})
export class MyComponent {
  constructor(
    private router: Router,
    private service: MyService,
    private cd: ChangeDetectorRef
  ) {}
}

// 修改后
@Component({...})
export class MyComponent {
  private router = inject(Router);
  private service = inject(MyService);
  private cd = inject(ChangeDetectorRef);
}
```

## 5. 图片 srcset 语法变化

```html
<!-- 修改前 -->

<!-- 修改后 -->
<img [srcset]="imageSrc" width="100%" height="auto">
```

## 6. CommonModule 导入变化

### NgModule 中

```typescript
// 修改前
import { CommonModule } from '@angular/common';

@NgModule({
  imports: [CommonModule]
})
export class MyModule { }
```

### Standalone 组件中

```typescript
// Angular 17 Standalone 组件
@Component({
  standalone: true,
  imports: [CommonModule],  // 直接导入
  template: `...`
})
export class MyComponent { }
```

---

## 7. 模块导入顺序

### 路由模块

```typescript
// 修改前
const routes: Routes = [
  { path: '', component: HomeComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)]
})
export class AppRoutingModule { }

// 修改后 - 可以直接使用 standalone 组件
const routes: Routes = [
  { path: '', component: HomeComponent }  // HomeComponent 可以是 standalone
];
```

## 8. postinstall 变化

### package.json

```json
// 修改前
"postinstall": "node scripts/ensure-babel-runtime-link.js"

// 修改后 - Angular 17 不再需要 ngcc
"postinstall": "node scripts/ensure-babel-runtime-link.js"
```

## 总结

| 变化类型 | 必须修改 | 建议修改 |
|---------|---------|---------|
| TypeScript 版本 | ✅ | - |
| Forms 类型 | ✅ | - |
| 新的控制流语法 | ❌ | ✅ |
| inject() 函数 | ❌ | ✅ |
| Standalone 迁移 | ❌ | - |

**注意**: Angular 17 完全向后兼容，旧的语法仍然可以使用。建议逐步迁移新特性，无需一次性全部修改。
