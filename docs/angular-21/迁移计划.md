## 概述

将项目从 Angular 14 架构迁移到 Angular 21 现代化架构，目标实现 **Zoneless + Signals**。

**原则**:
- 每步独立、可回滚
- 不破坏现有功能
- 可以分批提交
- 渐进式改进

### 阶段 2: ChangeDetectionStrategy.OnPush 🔴

**优先级**: 🔴 最高（阻塞 Zoneless）
**难度**: ⭐⭐⭐ 中等
**状态**: 未开始

#### 目标
将所有 156 个 `ChangeDetectionStrategy.Default` 组件改为 `OnPush`。

#### 策略
1. 优先处理无复杂状态的简单组件
2. 中间件处理有 RxJS 订阅的组件
3. 最后处理有第三方库依赖的组件（D3, xterm 等）

#### 示例
```typescript
// 修改前
@Component({
  standalone: true,
  changeDetection: ChangeDetectionStrategy.Default,
  // ...
})

// 修改后
@Component({
  standalone: true,
  changeDetection: ChangeDetectionStrategy.OnPush,
  // ...
})
```

#### 注意事项
- OnPush 组件需要在以下情况触发更新：
  - Input 引用变化
  - 手动调用 `markForCheck()`
  - 使用 signals/effects 自动触发

### 阶段 3: Angular 21 新语法 🔴

**优先级**: 🟡 中
**难度**: ⭐ 简单
**状态**: 未开始

#### 3.1 移除 standalone: true
Angular 21+ standalone 是默认行为，不需要显式声明：

```typescript
// Angular 21+ 不需要
@Component({
  standalone: true,  // ❌ 可以删除
  // ...
})

// Angular 21+ 只需要
@Component({
  // standalone: true 不需要，默认为 true
  // ...
})
```

#### 3.2 host 对象替代 @HostBinding
```typescript
// 修改前
@HostBinding('class') componentCssClass;

// 修改后
host: { class: 'app-root' },
```

#### 3.3 styleUrl 替代 styleUrls
```typescript
// 修改前
styleUrls: ['./component.scss']

// 修改后
styleUrl: './component.scss'
```

#### 3.4 函数式 pipe 替代
```typescript
// 修改前
import { Pipe, PipeTransform } from '@angular/core';
@Pipe({ name: 'myPipe' })
export class MyPipe implements PipeTransform {
  transform(value: string): string { return value.toUpperCase(); }
}

// 修改后 - 直接用纯函数
export const myPipe = (value: string): string => value.toUpperCase();
```

### 阶段 4: Services 层 Signals 化 🔴

**优先级**: 🟡 中
**难度**: ⭐⭐⭐ 中等
**状态**: 未开始

#### 目标
将服务层的状态管理从 RxJS 转为 Signals。

#### 示例
```typescript
// 修改前 - RxJS
@Injectable({ providedIn: 'root' })
export class SettingsService {
  private settingsSubject = new BehaviorSubject<Settings>(defaultSettings);
  settings$ = this.settingsSubject.asObservable();

  updateSettings(settings: Settings) {
    this.settingsSubject.next(settings);
  }
}

// 修改后 - Signals
@Injectable({ providedIn: 'root' })
export class SettingsService {
  private _settings = signal<Settings>(defaultSettings);
  settings = this._settings.asReadonly();

  updateSettings(settings: Settings) {
    this._settings.set(settings);
  }
}
```

**验证**:
```bash

ng test
```

### 任务 2: 启用 provideZonelessChangeDetection

**文件: src/main.ts**

需要在所有其他 providers 之后添加：
```typescript
import { provideZonelessChangeDetection } from '@angular/core';

providers: [
  // ... 其他 providers
  provideZonelessChangeDetection(),
]
```

### 任务 3: 组件改为 OnPush

**原则**:
- 每个组件都应该能在 OnPush 模式下工作
- 使用 signals 作为状态管理
- 避免在组件内直接修改对象属性

**示例组件迁移**:

```typescript
// progress.component.ts - 迁移前
@Component({
  standalone: true,
  changeDetection: ChangeDetectionStrategy.Default,  // ❌
})
export class ProgressComponent implements OnInit, OnDestroy {
  visible = false;  // ❌ 可变状态
  error = null;
  routerSubscription: Subscription;

  ngOnInit() {
    this.progressService.state.subscribe((state) => {
      this.visible = state.visible;  // ❌ 直接赋值
      if (state.error && !this.error) {
        this.error = state.error;
      }
    });
  }
}

// progress.component.ts - 迁移后
@Component({
  standalone: true,
  changeDetection: ChangeDetectionStrategy.OnPush,  // ✅
})
export class ProgressComponent implements OnInit, OnDestroy {
  visible = signal(false);     // ✅ signal
  error = signal<any>(null);  // ✅ signal
  routerSubscription: Subscription;

  ngOnInit() {
    this.progressService.state.subscribe((state) => {
      this.visible.set(state.visible);  // ✅ signal.set()
      if (state.error && !this.error()) {
        this.error.set(state.error);    // ✅ signal.set()
      }
    });
  }
}
```

## Signals 使用指南

### signal()

```typescript
import { signal } from '@angular/core';

// 可写 signal
const count = signal(0);

// 读取
count();          // 获取值
count.set(1);     // 设置值
count.update(n => n + 1);  // 基于当前值更新
```

### computed()

```typescript
import { computed, signal } from '@angular/core';

const firstName = signal('John');
const lastName = signal('Doe');

// 只读派生状态
const fullName = computed(() => `${firstName()} ${lastName()}`);
```

### effect()

```typescript
import { effect, signal } from '@angular/core';

const count = signal(0);

// 副作用 - 谨慎使用
effect(() => {
  console.log('Count changed:', count());
});
```

### input() / output()

```typescript
import { input, output } from '@angular/core';

// 输入
items = input<Item[]>([]);
loading = input<boolean>(false);

// 输出
save = output<boolean>();
delete = output<number>();
```

### toSignal() (RxJS 互操作)

```typescript
import { toSignal } from '@angular/core/rxjs-interop';

export class MyComponent {
  private route = inject(ActivatedRoute);

  // Observable -> Signal
  params = toSignal(this.route.params);
  query = toSignal(this.route.queryParams, { initialValue: {} });
}
```

### rxResource() (异步资源)

```typescript
import { rxResource } from '@angular/core/rxjs-interop';

export class MyComponent {
  nodeId = input<string>();

  nodeConfig = rxResource({
    request: () => ({ id: this.nodeId() }),
    loader: (params) => this.nodeService.getConfig(params.id)
  });
}
```

## Zoneless 迁移指南

### 什么是 Zoneless

Zoneless 是 Angular 21 的实验性功能，允许应用在不依赖 zone.js 的情况下工作。

**优势**:
- 减少约 80-120KB 的 bundle 大小
- 更可预测的变更检测
- 更好的性能控制

**劣势**:
- 需要手动处理第三方库集成
- 需要将所有状态改为 signals
- 某些库的兼容性问题

### 迁移检查清单

```markdown
□ 移除 zone.js 导入
□ 更新 bootstrap 使用 provideZonelessChangeDetection
□ 所有组件改为 ChangeDetectionStrategy.OnPush
□ 组件内状态使用 signal()
□ 服务层状态使用 signal()
□ Observable 转换为 toSignal()
□ 第三方库使用 afterRender 初始化
□ 测试套件更新
□ E2E 测试更新
```

### 常见问题解决

#### 1. 第三方库不更新视图

```typescript
import { inject, ApplicationRef } from '@angular/core';

// 手动触发变更检测
const appRef = inject(ApplicationRef);
appRef.tick();
```

#### 2. setTimeout/setInterval 不触发更新

```typescript
import { afterRender } from '@angular/core';

// 使用 afterRender 代替 setTimeout
afterRender(() => {
  // 执行一次性操作
});
```

#### 3. 事件处理不触发更新

```typescript
// 使用 signals 处理事件
const isOpen = signal(false);

onClick() {
  this.isOpen.set(true);  // ✅ 自动触发更新
}
```

## 风险识别与缓解

### 🔴 高风险

#### 1. D3.js 地图组件
**风险**: D3 直接操作 DOM，Zoneless 下可能不更新

**缓解**:
```typescript
import { afterRender, signal } from '@angular/core';

export class D3MapComponent {
  nodes = signal<Node[]>([]);

  constructor() {
    afterRender(() => {
      this.initD3();
    });
  }

  // 使用 signal 更新，D3 回调中调用
  updateNodes(nodes: Node[]) {
    this.nodes.set(nodes);
  }
}
```

#### 2. xterm.js 控制台
**风险**: xterm 直接操作 DOM

**缓解**: 使用 signal 管理状态，通过 ViewChild 操作终端

#### 3. ng2-file-upload
**风险**: 依赖 ZoneJS 进行进度更新

**缓解**: 使用 `toSignal()` 转换上传进度，或手动调用 `tick()`

### 🟡 中风险

#### 4. WebSocket 实时更新
**风险**: 可能不触发变更检测

**缓解**:
```typescript
// 使用 toSignal 包装
messages = toSignal(this.websocket.messages$, { initialValue: [] });
```

#### 5. 表单验证
**风险**: 验证状态可能不更新

**缓解**:
```typescript
formValue = toSignal(this.form.valueChanges, { initialValue: this.form.value });
isValid = computed(() => this.form.valid);
```

### 🟢 低风险

#### 6. 性能问题
**风险**: 过度使用 effects

**缓解**: 优先使用 `computed()`，只在必要时使用 `effect()`
