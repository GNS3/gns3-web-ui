## 概述

将项目从 Angular 14 架构迁移到 Angular 21 现代化架构，最终目标是实现 **Zoneless + Signals**。

**原则**:
- ✅ 每步独立、可回滚
- ✅ 不破坏现有功能
- ✅ 可以分批提交
- ✅ 渐进式改进

### 当前状态
- ✓ 所有组件已迁移为 standalone (253/253, 100%)
- ✓ 项目使用 Angular 21
- ✓ 仍在使用 ZoneJS
- ✓ 使用传统的 `@Input()`/`@Output()` 装饰器
- ✓ 大部分组件使用默认变更检测策略
- ✓ 部分服务使用 `NgZone`

### 目标状态
- 移除 ZoneJS 依赖
- 启用 Zoneless 变更检测 (`provideZonelessChangeDetection`)
- 使用 Signals API (`signal`, `computed`, `effect`, `input`, `output`)
- 所有组件使用 `OnPush` 变更检测策略
- 移除 `NgZone` 依赖

## Angular 21 核心技术

### 1. 自动迁移工具 (Schematics)

Angular 21 提供了自动迁移支持。在开始手动大规模修改前，建议尝试：

```bash
# 尝试自动迁移 Signal Inputs
ng generate @angular/core:signal-input-migration

# 尝试自动迁移 Signal Queries (@ViewChild -> viewChild)
ng generate @angular/core:signal-queries-migration
```

### 2. toSignal (Angular Core RxJS Interop)

**核心原则**: 使用 `toSignal` 将 Observable 转换为 Signal，实现自动变更检测。

```typescript
import { toSignal } from '@angular/core/rxjs-interop';

export class MyComponent {
  // 将 valueChanges 转为 Signal，自动触发更新
  nameValue = toSignal(this.nameControl.valueChanges, { initialValue: '' });

  // 使用 computed 派生验证状态
  isValid = computed(() => this.nameControl.valid);
}
```

**注意**: 在 Zoneless 模式下，**不需要**手动调用 `markForCheck()`。

### 3. linkedSignal (Angular 19+)

适用于"选中节点 -> 加载属性"的联动场景：

```typescript
import { linkedSignal } from '@angular/core';

selectedNodeId = input<string>();

// 当 selectedNodeId 变化时，nodeDetails 自动重置并触发新请求
nodeDetails = linkedSignal({
  source: this.selectedNodeId,
  computation: (id) => this.fetchNodeDetails(id),
  equal: (a, b) => a?.id === b?.id
});
```

### 4. rxResource (Angular 19+)

适用于"加载 -> 编辑 -> 提交"的声明式异步资源加载：

```typescript
import { rxResource } from '@angular/core/rxjs-interop';

export class NodeConfigComponent {
  nodeId = input<string>();
  private nodeService = inject(NodeService);

  // 自动处理 loading、error、data 状态
  nodeConfig = rxResource({
    request: () => ({ id: this.nodeId() }),
    loader: (params) => this.nodeService.getNodeConfig(params.id)
  });
}
```

模板中使用：
```html
@if (nodeConfig.isLoading()) {
  <mat-spinner />
} @else if (nodeConfig.error()) {
  <div>Error: {{ nodeConfig.error() | json }}</div>
}
```

### 5. afterRender / afterNextRender

用于 D3.js 等直接操作 DOM 的库集成：

```typescript
import { afterRender, inject, ChangeDetectorRef } from '@angular/core';

export class D3MapComponent {
  private cdr = inject(ChangeDetectorRef);
  private d3Initialized = false;

  constructor() {
    // 使用 afterRender 初始化 D3，确保 DOM 已就绪
    afterRender(() => {
      if (!this.d3Initialized) {
        this.initD3();
        this.d3Initialized = true;
      }
    });
  }

  // D3 回调中更新 Angular Signal 时，Zoneless 自动感知
  onDrag(event: D3DragEvent) {
    this.nodePosition.update(pos => ({ ...pos, x: event.x, y: event.y }));
  }
}
```

### 6. RxJS 与 Signals 共存策略

**原则**: **不要**为了用 Signal 而强行把所有 Observable 转为 Signal。

对于 WebSocket 实时流等场景：
- 保持 Observable 在服务层
- 在模板中使用 `async` 管道
- 或使用 `toSignal(stream$, { rejectErrors: true })`

```typescript
// 推荐：保持 Observable，用 async 管道
messages$ = this.websocketService.messages$;

// 或转换为 Signal
messages = toSignal(this.websocketService.messages$, { initialValue: [] });
```

### 7. 表单迁移正确方案

**旧方案 (Angular 18/19 过渡)**:
```typescript
ngOnInit() {
  this.form.valueChanges.subscribe(() => this.cd.markForCheck());
}
```

**Angular 21 正确方案**:
```typescript
export class MyFormComponent {
  form = new FormGroup({
    name: new FormControl(''),
  });

  // 用 toSignal 将 valueChanges 转为 Signal
  formValue = toSignal(this.form.valueChanges, { initialValue: this.form.value });

  // 用 computed 派生验证状态
  isValid = computed(() => this.form.valid);
}
```

### 8. 第三方库处理

如果移除 ZoneJS 后某些功能失效（如上传进度）：

```typescript
import { inject, ApplicationRef } from '@angular/core';

// 不需要重新引入 ZoneJS，手动触发一次全局变更检测
inject(ApplicationRef).tick();
```

## 风险识别与缓解

### 高风险

#### 1. D3.js 地图组件不更新
**风险描述**: D3.js 直接操作 DOM，在 zoneless 模式下需要特殊处理。

**缓解措施**:
- 使用 `afterRender` / `afterNextRender` 初始化 D3 容器
- 在 D3 回调中更新 Angular Signal，Zoneless 自动感知
- 如果更新非 Signal 状态，显式调用 `ChangeDetectorRef.markForCheck()`
- 充分测试地图交互（拖拽、缩放、节点操作）

#### 2. WebSocket 实时数据不更新
**风险描述**: WebSocket 获取实时更新可能不会触发变更检测。

**缓解措施**:
- 使用 `toSignal(websocket$, { rejectErrors: true })` 转换
- 或在 WebSocket 消息处理中调用 `markForCheck()`
- 模板中使用 `async` 管道

#### 3. 第三方库不兼容
**风险描述**: 某些第三方库可能依赖 ZoneJS。

**缓解措施**:
- 检查 `ng2-file-upload`, `angular-draggable-droppable`, `xterm` 等库
- 如有问题，使用 `inject(ApplicationRef).tick()` 手动触发变更检测

### 中风险

#### 4. 表单验证状态不更新
**风险描述**: 使用传统 ReactiveForms 但未适配 `toSignal` 可能导致验证状态不更新。

**缓解措施**:
- 使用 `toSignal(this.form.valueChanges)` 将表单状态转换为 Signal
- 使用 `computed(() => this.form.valid)` 派生验证状态
- 或迁移到 Angular Signal Forms（如果可用）

#### 5. 路由导航后视图不更新
**风险描述**: 路由参数变化可能不触发组件更新。

**缓解措施**:
- 使用 `toSignal(activatedRoute.params)` 包装路由参数
- 确保路由相关状态是 signals

### 低风险

#### 6. 性能回归
**风险描述**: 不当使用 signals 可能导致性能下降。

**缓解措施**:
- 建立性能基准
- 使用 Angular DevTools 进行性能分析
- 避免过度使用 `effect()`
- **帧率测试 (FPS)**: 使用 Chrome DevTools -> Rendering -> Frame Rendering Stats
  - 在 500+ 节点的拓扑图中拖拽，FPS 应保持在 60 左右
  - 目标: Scripting 耗时应下降 40% 以上

## 回滚计划

### 触发条件
如果出现以下情况，考虑回滚：
1. 核心功能严重损坏且无法快速修复

3. 大量用户反馈严重问题

1. **回滚代码** - `git revert <migration-commit-range>`
2. **恢复依赖** - `npm install zone.js`
3. **验证** - 运行测试套件，手动验证核心功能
