# 回滚计划 - Angular 21 现代化迁移

## 📋 概述

本文档定义了现代化迁移过程中的回滚策略和程序。确保在任何阶段出现问题时可以快速恢复到稳定状态。

---

## 🔴 紧急回滚触发条件

### 必须立即回滚的情况
1. **生产环境崩溃** - 应用无法正常启动或运行
2. **核心功能失效** - 主要功能（如拓扑编辑、节点管理）无法使用
3. **数据丢失风险** - 可能导致用户数据丢失的变更
4. **安全性问题** - 引入安全漏洞

### 建议回滚的情况
1. **性能严重下降** - 页面加载时间增加 > 50%
2. **大量回归错误** - > 10 个功能受影响
3. **团队 productivity 显著下降**

## 📊 分阶段回滚策略

### Phase 0: 准备阶段

**状态**: 已完成

**回滚**: 无需回滚（仅文档和基础设施）

---

### Phase 1: Standalone Components 试点

**风险等级**: 🟡 中

**回滚步骤**:
```bash
# 1. 切换到稳定分支
git checkout master

# 2. 重新安装依赖
npm install

# 3. 验证应用正常
npm start
```

**恢复时间**: ~5 分钟

---

### Phase 2: Signals 引入

**风险等级**: 🟡 中

**回滚步骤**:
```bash
# 1. 回滚代码更改
git checkout master -- src/app/

# 2. 清理构建缓存
rm -rf dist/ .angular/

# 3. 重新构建
npm run build
```

**恢复时间**: ~10 分钟

---

### Phase 3-8: 全面迁移

**风险等级**: 🔴 高

**回滚策略**:
- 每个 Phase 完成后创建 Git tag
- 保留 master 分支为稳定版本
- 使用 Feature Branch 进行隔离开发

## 🛠 回滚命令参考

### 快速回滚（推荐）
```bash
# 回滚到上一个稳定版本
git checkout <last-stable-tag>

# 或回滚到 master
git checkout master
```

### 选择性回滚
```bash
# 查看提交历史
git log --oneline

# 回滚特定文件
git checkout <commit-hash> -- path/to/file

# 创建回滚 commit
git revert <commit-hash>
```

### 完全重置（谨慎使用）
```bash
# 放弃所有本地更改
git reset --hard <commit-hash>

# 强制推送到远程（仅管理员）
git push --force-with-lease origin modernization/angular-21-migration
```

## 🔒 分支保护策略

### 主分支保护
```
master:   # 稳定分支，永不直接推送
upgrade/angular-21:  # 升级分支，需要 PR 合并
modernization/angular-21-migration:  # 迁移分支，开发专用
```

### 发布流程
```
开发分支 → PR → review → 合并到升级分支 → 测试 → 合并到 master → 发布
```
